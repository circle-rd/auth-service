# Quick start

## 1. Start the database

The dev Docker Compose file starts a PostgreSQL 17 container on port 5433:

```sh
docker compose -f docker-compose.dev.yml up -d postgres
```

## 2. Push the schema

`db:push` syncs the Drizzle schema directly to the database without generating migration files — the right approach for development:

```sh
pnpm db:push
```

## 3. Start the backend

```sh
pnpm dev
```

The server starts with pino-pretty logging and hot-reload via `tsx watch`. It is available at `http://localhost:3001`.

## 4. (Optional) Start the frontend

The Vue 3 admin SPA has a separate Vite dev server with HMR:

```sh
cd frontend
pnpm install
pnpm dev
```

Frontend: `http://localhost:5173` — Vite proxies API requests to the backend on port 3001.

For quick iteration you can use the backend directly on port 3001 — when `frontend-dist/` is present (after a `pnpm build`), the Vue SPA is served from there.

## 5. Create the first superadmin

Set `ADMIN_EMAIL` and `ADMIN_PASSWORD` in `.env` before starting the server. The bootstrap routine creates a superadmin on first boot if none exists:

```dotenv
ADMIN_EMAIL=admin@example.com
ADMIN_PASSWORD=changeme123
```

The bootstrap logic is in `src/bootstrap.ts` and runs automatically at server startup.

## 6. Verify

```sh
curl http://localhost:3001/health
# → {"status":"ok","version":"0.1.0"}
```

OIDC discovery document (requires the server to be running):

```sh
curl http://localhost:3001/api/auth/.well-known/openid-configuration
```

## Available scripts

| Script | Description |
|---|---|
| `pnpm dev` | Start backend with hot-reload |
| `pnpm build` | Build backend TypeScript + frontend Vue SPA |
| `pnpm build:server` | Build only the TypeScript backend |
| `pnpm build:frontend` | Build only the Vue frontend |
| `pnpm start` | Start the compiled production server |
| `pnpm db:generate` | Generate Drizzle migration files from schema changes |
| `pnpm db:migrate` | Apply pending migrations (production) |
| `pnpm db:push` | Push schema directly to DB (dev only) |
| `pnpm test` | Run unit tests with Vitest |
| `pnpm test:integration` | Run integration tests (requires a live DB) |
