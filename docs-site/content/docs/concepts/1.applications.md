# Applications

An **application** in auth-service is an OAuth 2.0 client registration combined with an access policy, an RBAC model, and subscription plans. Every application has a unique `slug` that doubles as its OAuth `client_id`.

## Application schema

| Field | Type | Default | Description |
|---|---|---|---|
| `id` | UUID | auto | Primary key |
| `name` | string | required | Display name (max 100 chars) |
| `slug` | string | required | URL-safe identifier, lowercase alphanumeric + hyphens. Used as `client_id`. |
| `description` | string | — | Optional description (max 500 chars) |
| `isActive` | boolean | `true` | Whether the application is active |
| `isPublic` | boolean | `false` | Public (PKCE-only) vs confidential client |
| `skipConsent` | boolean | `false` | Skip the OAuth consent screen |
| `isMfaRequired` | boolean | `false` | Require MFA for all users of this app |
| `allowRegister` | boolean | `true` | Allow new-user self-registration via this app |
| `allowedScopes` | string[] | `["openid","profile","email"]` | Scopes the app is permitted to request |
| `redirectUris` | string[] | `[]` | Allowed OAuth redirect URIs |
| `url` | string | — | Application URL (informational) |
| `icon` | string | — | Icon URL (displayed in the admin SPA) |
| `enabledSocialProviders` | string[] \| null | `null` | Per-app social providers. `null` inherits all globally configured providers. |

## Public vs confidential clients

When `isPublic` is `true`:

- No client secret is generated.
- PKCE (`code_challenge` / `code_verifier`) is **required** on every authorization request.
- `token_endpoint_auth_method` is set to `none` in the OAuth client record.

When `isPublic` is `false` (default):

- A random 32-byte hex secret is generated at creation time.
- The secret is stored hashed (SHA-256 base64url) and returned **once** in the creation response.
- The client must authenticate with `client_id` + `client_secret` at the token endpoint.

## Bootstrap macros

When a new application is created via `POST /api/admin/applications`, auth-service automatically:

1. Creates two default roles: `user` (default) and `admin`.
2. Creates a default `free` subscription plan.
3. Assigns all existing `superadmin` users to the application with the `admin` role and the `free` plan.

## Admin API endpoints

All application management endpoints require an `admin` or `superadmin` session.

### List applications

```
GET /api/admin/applications
```

Returns all applications ordered by creation date.

```json
{
  "applications": [
    {
      "id": "...",
      "name": "My App",
      "slug": "my-app",
      "isActive": true,
      "isPublic": false,
      ...
    }
  ]
}
```

### Create an application

```
POST /api/admin/applications
```

```json
{
  "name": "My App",
  "slug": "my-app",
  "description": "Optional description",
  "isPublic": false,
  "allowedScopes": ["openid", "profile", "email", "roles"],
  "redirectUris": ["https://app.example.com/callback"]
}
```

The response includes a `clientSecret` field that contains the plaintext secret. Store it immediately — it cannot be retrieved afterwards.

### Get an application

```
GET /api/admin/applications/:id
```

### Update an application

```
PATCH /api/admin/applications/:id
```

All fields except `slug` and `isPublic` can be updated. If `skipConsent` or `redirectUris` change, the underlying BetterAuth `oauthClient` record is updated in the same transaction.

### Delete an application

```
DELETE /api/admin/applications/:id
```

Deletion is blocked if the application has active user subscriptions (`APP_004`). All dependent records (roles, permissions, user access, consumption data) are cascade-deleted.

### Rotate client secret

```
POST /api/admin/applications/:id/rotate-secret
```

Generates a new random secret, hashes it, stores it, and returns the plaintext value once. The old secret is invalidated immediately.

## Managing user access

### Grant access to a user

```
POST /api/admin/applications/:id/users
```

```json
{
  "userId": "...",
  "roleId": "..."   // optional — assigns a specific role immediately
}
```

### List users with access

```
GET /api/admin/applications/:id/users
```

### Update a user's access

```
PATCH /api/admin/applications/:id/users/:userId
```

```json
{
  "isActive": true,
  "roleId": "...",
  "subscriptionPlanId": "..."
}
```

### Revoke access

```
DELETE /api/admin/applications/:id/users/:userId
```
