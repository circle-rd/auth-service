# Installation

## Prerequisites

- **Node.js 22+** and **pnpm 9+**
- **Docker** and **Docker Compose** (for the PostgreSQL database)
- **PostgreSQL 17** (provided by the included Docker Compose files)

## Clone and install

```sh
git clone https://github.com/circle-rd/auth-service.git
cd auth-service
pnpm install
```

The frontend workspace is a separate pnpm project under `frontend/`. Installing at the root does **not** install frontend dependencies automatically — see [Quick Start](/docs/getting-started/quick-start) for the full workflow.

## Environment variables

Copy the example file and set the required values:

```sh
cp .env.example .env
```

Generate a secure random secret for `BETTER_AUTH_SECRET`:

```sh
openssl rand -base64 32
```

Minimum required variables for a development environment:

```dotenv
PORT=3001
HOST=0.0.0.0
NODE_ENV=development

BETTER_AUTH_SECRET=<output of openssl rand -base64 32>
BETTER_AUTH_URL=http://localhost:3001

DATABASE_URL=postgres://auth:auth@localhost:5433/auth

# Docker Compose postgres credentials (used by docker-compose.dev.yml)
POSTGRES_USER=auth
POSTGRES_PASSWORD=auth
POSTGRES_DB=auth
POSTGRES_PORT=5433
```

The dev compose file exposes PostgreSQL on port **5433** to avoid conflicts with a local PostgreSQL instance on 5432.

## Database connection

Auth Service uses [Drizzle ORM](https://orm.drizzle.team/) with the `postgres` driver. The `DATABASE_URL` environment variable must be a valid PostgreSQL connection URL:

```
postgres://<user>:<password>@<host>:<port>/<database>
```

In the Docker Compose setup the hostname is `postgres` (the service name); externally it is `localhost:5433`.
