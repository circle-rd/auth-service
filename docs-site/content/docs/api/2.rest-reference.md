# REST API reference

## Authentication

Unless noted as public, all endpoints require an authenticated session. Admin endpoints require the `admin` or `superadmin` global role. Pass the BetterAuth session cookie (set on sign-in) or include the `Authorization: Bearer <token>` header.

## System

| Method | Path | Auth | Description |
|---|---|---|---|
| `GET` | `/health` | public | Health check. Returns `{"status":"ok","version":"<version>"}` |

## BetterAuth (OAuth 2.1 / OIDC)

All BetterAuth-managed routes are prefixed with `/api/auth/`. Full documentation is at [OAuth 2.1 / OIDC endpoints](/docs/concepts/oauth-oidc).

| Method | Path | Auth | Description |
|---|---|---|---|
| `ALL` | `/api/auth/*` | varies | BetterAuth handler — login, register, OAuth2, OIDC, MFA, passkeys |
| `GET` | `/api/auth/.well-known/openid-configuration` | public | OIDC discovery document |
| `GET` | `/api/auth/.well-known/oauth-authorization-server` | public | RFC 8414 metadata |
| `GET` | `/api/auth/jwks` | public | JSON Web Key Set |
| `GET/POST` | `/api/auth/oauth2/authorize` | public | Authorization endpoint |
| `POST` | `/api/auth/oauth2/token` | public | Token endpoint |
| `GET` | `/api/auth/oauth2/userinfo` | Bearer | UserInfo endpoint |
