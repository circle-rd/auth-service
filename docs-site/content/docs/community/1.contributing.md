# Contributing

Contributions are welcome. Please read the [CONTRIBUTING.md](https://github.com/circle-rd/auth-service/blob/main/CONTRIBUTING.md) in the repository for detailed guidelines.

## Development workflow

```sh
git clone https://github.com/circle-rd/auth-service.git
cd auth-service
pnpm install
cp .env.example .env
# Edit .env
docker compose -f docker-compose.dev.yml up -d postgres
pnpm db:push
pnpm dev
```

## Running tests

Unit tests (no database required):

```sh
pnpm test
```

Integration tests (require a live PostgreSQL — provided by the dev compose file):

```sh
pnpm test:integration
```

## Code style

- TypeScript strict mode (`"strict": true` in `tsconfig.json`)
- ESM only (`"type": "module"` in `package.json`)
- No default exports from route files — all exports are named functions

## Submitting changes

1. Fork the repository and create a branch from `main`.
2. Make your changes with appropriate test coverage.
3. Ensure `pnpm test` and `pnpm test:integration` pass.
4. Open a pull request against `main`.
