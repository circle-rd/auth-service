# Organizations

Auth Service includes first-class support for **organizations** (companies / tenants) powered by BetterAuth's `organization()` plugin. Organizations are designed for a two-tier distribution model:

| Tier | Actor | Typical role |
|---|---|---|
| 1 | Reseller / distributor | Creates and owns organizations representing their end-clients |
| 2 | Integrator | Member of one or more organizations; manages IT resources scoped to those organizations |

## Organization schema

| Field | Type | Description |
|---|---|---|
| `id` | string | Primary key |
| `name` | string | Display name (max 100 chars) |
| `slug` | string | Unique URL-safe identifier (lowercase alphanumeric + hyphens) |
| `logo` | string | Optional logo URL |
| `metadata` | JSON object | Arbitrary key-value metadata |
| `createdAt` | timestamp | Creation date |

## Member roles

Every member of an organization holds one of three roles:

| Role | Description |
|---|---|
| `owner` | Full control over the organization and its members |
| `admin` | Can manage members and invitations |
| `member` | Read-only membership |

## The `org_id` claim

When a client requests the `org` scope during an OAuth 2.0 authorization flow, the access token includes an `org_id` claim set to the user's **active organization ID** (`session.activeOrganizationId`):

```json
{
  "org_id": "org_01hwxyz..."
}
```

Backend services can use this claim to:

- Scope database queries to the organization's data.
- Enforce CASL abilities with `{ org_id }` conditions.
- Isolate resources between tenants without additional requests to auth-service.

## Access control

Only users with the global role `admin` or `superadmin` can create organizations and manage members. Regular `user`-role accounts can belong to organizations as members but cannot create them.

## Admin API endpoints

### List organizations

```
GET /api/admin/organizations
```

Supports pagination and filtering:

| Query parameter | Default | Description |
|---|---|---|
| `page` | `1` | Page number |
| `limit` | `20` | Results per page (max 100) |
| `search` | — | Filter by name or slug (case-insensitive, partial match) |
| `sortBy` | `name` | Sort field: `name`, `slug`, or `createdAt` |
| `sortOrder` | `asc` | `asc` or `desc` |

Response includes a `memberCount` for each organization.

### Create an organization

```
POST /api/admin/organizations
```

```json
{
  "name": "Acme Corp",
  "slug": "acme-corp",
  "logo": "https://example.com/logo.png",
  "metadata": { "region": "eu-west" }
}
```

### Get an organization

```
GET /api/admin/organizations/:id
```

Returns the organization with its full member list.

### Delete an organization

```
DELETE /api/admin/organizations/:id
```

### List members

```
GET /api/admin/organizations/:id/members
```

### Add a member

```
POST /api/admin/organizations/:id/members
```

```json
{
  "userId": "...",
  "role": "member"
}
```

### Update a member's role

```
PATCH /api/admin/organizations/:id/members/:memberId/role
```

```json
{
  "role": "admin"
}
```

### Remove a member

```
DELETE /api/admin/organizations/:id/members/:userId
```

### List invitations

```
GET /api/admin/organizations/:id/invitations
```

### Send an invitation

```
POST /api/admin/organizations/:id/invitations
```

```json
{
  "email": "newmember@example.com",
  "role": "member"
}
```

Sends an invitation email via BetterAuth's invitation flow. Requires SMTP to be configured for the email to be delivered.

### Cancel an invitation

```
DELETE /api/admin/organizations/:id/invitations/:invitationId
```
