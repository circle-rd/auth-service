# Custom templates

Auth Service ships built-in HTML templates for the `login`, `register`, `verify-email`, and `select-org` pages. You can override any of these templates globally or per application by mounting a volume at the directory pointed to by `TEMPLATES_DIR`.

## Resolution order

For a given page (e.g. `login`) and application slug (e.g. `my-app`), the template resolver checks candidates in this order:

1. `<TEMPLATES_DIR>/<appSlug>/login.html` — per-application override
2. `<TEMPLATES_DIR>/default/login.html` — global override
3. Built-in template shipped with auth-service — fallback

If none of the first two candidates exist, the built-in template is used. If the built-in template is also missing (which should not happen in a normal installation), the Vue SPA index.html is served instead.

## Template variables

Templates receive simple `{{VAR}}` substitutions:

| Variable | Description |
|---|---|
| `{{ACTION_URL}}` | Form action URL for the current page |
| `{{REDIRECT_TO}}` | The `redirectTo` query parameter forwarded from the OAuth flow |
| `{{APP_SLUG}}` | The `client_id` / application slug |
| `{{AUTH_URL}}` | The `BETTER_AUTH_URL` value (base URL of auth-service) |
| `{{ERROR_MESSAGE}}` | Error message from a previous failed attempt (may be empty) |
| `{{OAUTH_QUERY}}` | Full OAuth query string (including BetterAuth signature) for resuming the authorization flow |
| `{{ALLOW_REGISTER}}` | `"true"` or `"false"` — whether new-user registration is permitted |
| `{{SOCIAL_PROVIDERS_JSON}}` | JSON array of enabled social provider slugs, e.g. `["google","github"]` |
| `{{LOGIN_URL}}` | Pre-computed login URL with OAuth query string preserved |
| `{{REGISTER_URL}}` | Pre-computed register URL (empty string when `allowRegister` is false) |
| `{{ORGANIZATIONS_JSON}}` | JSON array of user's organizations (select-org page only) |

## Docker volume mount

```yaml
services:
  auth-service:
    image: ghcr.io/circle-rd/auth-service:latest
    environment:
      TEMPLATES_DIR: /templates
    volumes:
      - ./my-templates:/templates
```

Directory structure example:

```
my-templates/
├── default/
│   ├── login.html          # global override for all apps
│   └── register.html
└── my-app/
    └── login.html          # override only for the "my-app" application
```

## Minimal login template example

```html
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <title>Sign in</title>
</head>
<body>
  {{#if ERROR_MESSAGE}}
  <p class="error">{{ERROR_MESSAGE}}</p>
  {{/if}}

  <form method="POST" action="{{ACTION_URL}}">
    <input type="hidden" name="callbackURL" value="{{REDIRECT_TO}}" />
    <input type="hidden" name="oauth_query" value="{{OAUTH_QUERY}}" />
    <label>Email <input type="email" name="email" required /></label>
    <label>Password <input type="password" name="password" required /></label>
    <button type="submit">Sign in</button>
  </form>

  {{#if ALLOW_REGISTER}}
  <a href="{{REGISTER_URL}}">Create an account</a>
  {{/if}}
</body>
</html>
```

::callout{icon="i-lucide-info"}
The `oauth_query` hidden field is required in the login form. BetterAuth uses it to resume the OAuth authorization flow after a successful sign-in. Without it, the user will be authenticated but will not be redirected back to the client application.
::
