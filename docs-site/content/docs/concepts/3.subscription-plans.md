# Subscription plans

Each application can have multiple **subscription plans**. A plan carries an arbitrary `features` JSON object that is injected as a `features` claim in JWT access tokens, allowing downstream services to gate functionality without querying auth-service.

## Plan schema

| Field | Type | Default | Description |
|---|---|---|---|
| `id` | UUID | auto | Primary key |
| `applicationId` | UUID | required | Application this plan belongs to |
| `name` | string | required | Plan name (unique per application, max 64 chars) |
| `description` | string | — | Optional description |
| `stripeProductId` | string | — | Stripe Product ID (auto-created when Stripe is configured) |
| `features` | JSON object | `{}` | Arbitrary feature flags, e.g. `{"maxProjects": 10, "apiAccess": true}` |
| `isDefault` | boolean | `false` | Automatically assigned when a user is granted access to the app and no plan is specified |

Only one plan per application can have `isDefault = true`.

## Plan prices

A plan can have multiple **prices** (monthly, annual, one-time):

| Field | Type | Default | Description |
|---|---|---|---|
| `id` | UUID | auto | Primary key |
| `planId` | UUID | required | Parent plan |
| `name` | string | required | Price label, e.g. `Monthly` |
| `amount` | numeric | required | Amount in smallest currency unit (e.g. cents) |
| `currency` | string | `usd` | ISO 4217 currency code |
| `interval` | `"month"` \| `"year"` \| `"one_time"` | `month` | Billing interval |
| `stripePriceId` | string | — | Stripe Price ID (auto-created when Stripe is configured) |

## Stripe integration

When `STRIPE_SECRET_KEY` is set:

- Creating a plan via `POST /api/admin/applications/:appId/plans` **automatically creates a Stripe Product** if no `stripeProductId` is provided.
- Creating a price via `POST /api/admin/applications/:appId/plans/:planId/prices` **automatically creates a Stripe Price** linked to the product.

The Stripe Product name is `<AppName> — <PlanName>`.

## Admin API endpoints

### List plans with prices

```
GET /api/admin/applications/:appId/plans
```

Returns all plans with their associated prices:

```json
{
  "plans": [
    {
      "id": "...",
      "name": "pro",
      "features": { "maxProjects": 50, "apiAccess": true },
      "isDefault": false,
      "prices": [
        { "name": "Monthly", "amount": 1500, "currency": "usd", "interval": "month" }
      ]
    }
  ]
}
```

### Create a plan

```
POST /api/admin/applications/:appId/plans
```

```json
{
  "name": "pro",
  "description": "Professional tier",
  "features": { "maxProjects": 50, "apiAccess": true },
  "isDefault": false
}
```

### Update a plan

```
PATCH /api/admin/applications/:appId/plans/:planId
```

### Delete a plan

```
DELETE /api/admin/applications/:appId/plans/:planId
```

Blocked if any active user is subscribed to the plan (`SUB_003`).

### Create a price

```
POST /api/admin/applications/:appId/plans/:planId/prices
```

```json
{
  "name": "Monthly",
  "amount": 1500,
  "currency": "usd",
  "interval": "month"
}
```

### Delete a price

```
DELETE /api/admin/applications/:appId/plans/:planId/prices/:priceId
```

## Assigning a subscription to a user

Subscriptions are managed via the user access endpoint:

```
PATCH /api/admin/applications/:appId/users/:userId
```

```json
{
  "subscriptionPlanId": "<plan-uuid>",
  "isActive": true
}
```

An optional expiry date can be set on the underlying `userSubscriptions` record via the admin consumption routes.

## Feature claims in tokens

When a client requests the `features` scope, the access token contains:

```json
{
  "features": { "maxProjects": 50, "apiAccess": true }
}
```

The value is the `features` JSON from the user's current subscription plan. If no plan is assigned, `features` is `{}`.
